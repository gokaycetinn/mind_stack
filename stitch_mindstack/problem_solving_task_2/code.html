SYS-DESIGN-402 Rate Limiting

The Leaking Bucket

Your API rate limiter is currently dropping valid requests during sudden traffic spikes. The engineering lead points out that the current implementation uses a Fixed Window Counter.

You need an algorithm that allows for bursts of traffic but processes them at a steady rate to prevent overwhelming the backend services.
